# Resource Activation Timeout — Mixed Mode (EUtranCellFDD / EUtranCellTDD / NRSectorCarrier / NodeBLocalCell)

## Overview

The `Resource Activation Timeout` alarm is raised when activation of resources
hangs or when the cell is activated and resource allocation fails. In the Mixed
Mode scenario, this is caused by a missing signal from the requested resource,
by resources that fail during node startup, or by resources that are disabled
during startup.

**Important note on software fallback:** When an older software configuration
version is restored on the node in fallback, cell resource allocation takes
more time than usual and this alarm is raised. The alarm resolves automatically
after five minutes as the new configuration file (generated by the newer
software version) is replaced by one readable by the older version. No action
required in this specific case.

**Important note on antenna configuration:** Ensure correct settings (not 0)
for `noOfRxAntennas` and `noOfTxAntennas` when Dual-Antenna Downlink,
Transmit Data Cloning, 4x2 Quad Antenna, Octal Antenna Uplink, or Beamforming
Performance Package features are ENABLED. For Massive MIMO capable AAS, both
must be set to 0.

Managed Object: `EUtranCellFDD`, `EUtranCellTDD`, `NRSectorCarrier`, `NodeBLocalCell`
Alarm type: Primary

## Preconditions

- Resource Activation Timeout alarm verified active
- Additional Text confirms Mixed Mode scenario
- No higher-severity correlated alarms pending resolution
- NMS access confirmed with ability to lock and unlock cell MOs

## Resolution Steps

1. Check for correlated alarms and resolve alarms with higher severity first; for detailed instructions on identifying correlated alarms and determining their resolution order, see Check for Correlated Alarms in the NMS Fault Management guide.
2. Lock the cell using the EUtranCellFDD, EUtranCellTDD, or NbIotCell MO (set administrativeState to LOCKED).
3. Unlock the cell (set administrativeState to UNLOCKED) and wait 2 minutes for activation to complete.
4. If the alarm remains: ensure that the baseband and radio units are enabled and unlocked; for instructions on how to unlock a unit, see Manage Hardware Equipment or Unlock Board depending on the library in use.
5. Lock the cell again using EUtranCellFDD or EUtranCellTDD or NbIotCell MO.
6. Unlock the cell again and wait 2 minutes.
7. If the alarm remains: restart the radio unit; for more information, see Manage Hardware Equipment or Restart Board depending on the library in use.
8. If the alarm remains: restart the node; for more information, see Restart Node or Emergency Recover Node on Site depending on the library in use.
9. If the alarm remains: consult the next level of maintenance support.
